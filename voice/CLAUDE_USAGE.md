# Claude Usage Log

How Claude (Claude Code and the Claude API) is used to build this project. Append-only:
every milestone gets one dated entry at the tail, written by the `usage-log` agent
(`.claude/agents/usage-log.md`, pinned to Haiku; the thinking arrives in the dispatch
payload). The summary block below is regenerated from the entries by a Sonnet-level
ROLLUP pass and is the judge-facing overview.

Note that Claude is not only a build tool here: the product itself runs on Claude (the
assistant's replies are generated by Claude Haiku inside `web/server.py`).

**Two roles for Claude and Codex.** Claude Haiku is part of the product, generating live
spoken replies and bounded classification and tool-use decisions. Claude Code and Codex
built, operated, tested, and independently packaged the system for review across 22
logged milestones.

**Build-side operating model:** a read-only lead coordinates six single-writer lanes
(web-core, voice-ui, speech-svc, pod-ops, docs, usage-log) with deliberate model routing,
frozen or server-first WebSocket contracts, and lead inspection of every worker diff.
Agent-team waves, scripted workflows, plan mode, project memory, guard hooks, browser and
Hugging Face MCP access, source-code reading, research synthesis, remote diagnosis, and
specialized review skills were used where each fit. The team also recorded and corrected
a concurrent-writer collision, turning the incident into a stronger serialization rule.

**Product progression:** Claude-assisted work delivered the standalone editorial UI,
latency instrumentation, sentence-streaming TTS, barge-in cancellation, speculative LLM
start, voice and generation controls, durable session history, scoped multi-client
operator views, and a live authenticated demo. Infrastructure moved from a rented GPU pod to a
self-hosted split topology with durable service and tunnel launch recipes. A
field-debugging loop then fixed iPhone audio capture, instrumented and labeled 29 real
clips, derived an evidence-based noise gate, and scoped ASR hints per user. Investigation
repeatedly crossed abstraction layers, from browser computed styles and WebSocket
concurrency semantics to installed decoder internals, GPU compatibility, process
isolation, DNS, TLS, and service lifecycle behavior.

**Voice-safety differentiator:** the overnight agent team exposed ASR per-token
confidence, fused it with consequence severity for deterministic command confirmation,

announcements with alert arbitration, and delivered hands-busy protocol navigation plus
addressed-speech detection. Real operator labels now also drive low-confidence and
degenerate-transcript rejection, with explicit confirmation and stop exemptions. Claude
Haiku handles constrained language judgments, while safety-critical execution,
confirmation, cancellation, transcript visibility, and speculation commit gates remain
deterministic.

**Evidence discipline:** milestones were checked with local smoke suites, pytest,
