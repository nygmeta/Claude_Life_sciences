# Lab Agent — Web Console

**Just double-click `index.html`.** It is a single self-contained file: no build, no npm,
no server, no API key, no internet. Demo mode runs entirely from fixtures generated by real
backend runs.

The visible half of the Lab Agent: conversation, pipeline, compiled workflow, validation
verdict, robot routing, and audit trail. **Zero build step** — plain HTML + JS, no npm.

## Run it

```bash
# from the repo root
python -m http.server 8090 --directory web
# open http://localhost:8090
```

Opening `index.html` directly with `file://` also works in demo mode.

## Two modes

**Demo** (default) — runs entirely from `fixtures.js`, **no backend, no API key, no network**.
The fixtures were generated from real backend runs, so nothing on screen is invented.
Conference Wi-Fi cannot break this. Use it on stage.

**Live** — calls the API at `http://localhost:8000`. Start the backend first:

```bash
uvicorn app.main:app --reload
```

Then click **Live** and type a request. It reads `GET /adapters` so the routing panel
reflects each platform's *actual* declared capabilities.

## What's on screen

| Panel | Shows |
|---|---|
| **Conversation** | Scientist turns, agent replies, clarification questions |
| **Pipeline** | Intent → Plan → Workflow → Validate → Adapter → Execute, lighting up per stage (red on a validation failure) |
| **Plate** | A live 96-well plate; wells fill as operations execute |
| **Safety & validation** | Pass/blocked verdict, every issue with its rule name, plus the assumptions read back to the scientist |
| **Robot routing** | Both platforms, their declared capabilities, and **why** each accepts or refuses this workflow |
| **Audit trail** | Timestamped request → plan → compile → validate → execute, with the generated artifact |

## The two demo scenarios

**1 · ELISA** — asks for the analyte and sample count (never guesses them) → **catches a
400 µL transfer into a 300 µL well and blocks it** → accepts the correction → runs on
Opentrons. The Echo panel refuses it throughout (no `incubate`, volumes 20× over its max).

**2 · Dilution** — a 5-point nanoliter series compiles to an Echo picklist and runs.
The Opentrons panel refuses it (sub-microliter is below a tip-based floor).

Together these are the point: **same IR, same validator, opposite verdicts** — from
declared capabilities, not special-casing.

## Embedding in another page

The console is self-contained, so the simplest integration is an iframe:

```html
<iframe src="http://localhost:8090/index.html" style="width:100%;height:900px;border:0"></iframe>
```

To drive it from an existing voice pipeline instead, skip the UI and POST straight to the
backend — `POST /session/message` returns `reply` (speak this), plus `plan`, `workflow`,
`validation`, and `state`. The console is one consumer of that API, not a dependency of it.

## Regenerating fixtures

`fixtures.js` is generated from real backend runs. If you change an SOP, inventory, or a
safety rule, regenerate it so the demo keeps matching reality (see the snippet in the repo
root README, or just re-run the scenarios against a live backend in Live mode).
